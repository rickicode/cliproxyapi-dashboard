# AGENTS: Generated Code & Do-Not-Edit Rules

## TL;DR — What NOT to Edit

### 🔴 Critical: Never Touch These Paths
```
dashboard/src/generated/prisma/              ← Prisma Client (regenerate with: npm run prisma:generate)
dashboard/.next/                             ← Next.js build (regenerate with: npm run build)
dashboard/next-env.d.ts                      ← Next.js types (auto-generated)
docs/ENV.md (lines 3-23)                     ← Auto-generated environment docs
docs/CONTRIBUTING.md (lines 31-43)           ← Auto-generated script docs
```

### ✅ What You CAN Edit
- `dashboard/prisma/schema.prisma` — Edit schema, then run `npm run prisma:generate`
- Any `.tsx`, `.ts` files in `dashboard/src/` (except `src/generated/`)
- Any `.ts` files in `dashboard/lib/`, `dashboard/app/`, `dashboard/components/`

---

## How to Recognize Generated Files (When in Doubt)

### Header Markers (Copy/Paste These Patterns)
```
/* !!! This is code generated by X. Do not edit directly. !!! */
// @generated
// AUTO-GENERATED
// GENERATED FILE
```

### ESLint/Lint Suppressions
```
/* eslint-disable */
// biome-ignore-all lint: generated file
// @ts-nocheck
```

### Directory Names That Signal "Generated"
- `.next/`, `dist/`, `build/`, `out/`
- `src/generated/`, `__generated__/`

## Generation Tools & Recovery

| Tool | Problem | Solution |
|------|---------|----------|
| **Prisma Client** | Stale or missing types | `cd dashboard && npm run prisma:generate` |
| **Next.js Build** | `.next/` missing or broken | `cd dashboard && npm run build` |
| **Type Check Fails** | Generated types out of sync | `cd dashboard && npm run typecheck` (auto-runs prisma:generate) |

## Checklist: Before Committing

```
❌ Don't commit: dashboard/src/generated/prisma/*
❌ Don't commit: dashboard/.next/*
✅ Do regenerate after schema change: npm run prisma:generate
✅ Do regenerate before build: npm run build
```

---

## Anti-Patterns (From ANTI_PATTERNS.md)

**FORBIDDEN**:
- Editing `src/generated/prisma/internal/*` — wiped on next `prisma generate`
- Committing stale Prisma output after schema changes
- Modifying `.next/` contents — overwritten on build
- Hand-editing content between `AUTO-GENERATED:*` markers in docs

**REQUIRED**:
- Always run `npm run prisma:generate` after schema changes
- Treat schema as source of truth, not generated client code
- Regenerate on dependency updates or schema version bumps

## When Something Breaks

### "Cannot find module `@/generated/prisma`"
```bash
cd dashboard
npm run prisma:generate
npm run typecheck
```

### "Type `User` is not assignable to type `User`"
→ Stale Prisma types in `.next/` cache
```bash
cd dashboard
rm -rf .next src/generated
npm run build
```

### "Schema.prisma has errors after editing"
→ Run generator to validate and generate new types
```bash
cd dashboard
npm run prisma:generate
```

## Questions for AGENTS

**Q: "Can I edit this file?"**  
A: Check if it's in the **Critical Paths** list above. If yes → NO. If no → probably yes, but grep the file for `@generated` or `DO NOT EDIT` comments.

**Q: "Will my edits survive a rebuild?"**  
A: If the file is under `src/generated/` or `.next/` → NO. Run the appropriate regeneration command (see table above).

**Q: "I changed the schema but types are breaking."**  
A: Run `npm run prisma:generate` to sync generated types with schema.

**Q: "Can I commit generated files?"**  
A: NO. Check `.gitignore`:
```
.next/
/src/generated/
out/
build/
dist/
```
These should never be staged.

## Speed Bump: Regeneration Commands

Copy these one-liners when you modify schemas, styles, or configs:

```bash
# After editing prisma/schema.prisma:
cd dashboard && npm run prisma:generate

# Before deploying or building:
cd dashboard && npm run build

# After editing dependencies or major version bumps:
cd dashboard && rm -rf .next src/generated node_modules && npm install && npm run build

# Full typecheck (includes prisma:generate):
cd dashboard && npm run typecheck
```
